# Test Case: Dynamic Properties

## Test Case Attributes

| Attribute | Value |
| --- | --- |
| Test Case ID | TC-DEMOQA-3-2 |
| Title | Verify delayed enable, visibility, and dynamic style change |
| Objective | Validate that dynamic properties update correctly after their delayed triggers. |
| Requirement | Button becomes enabled, delayed button becomes visible, and color-change element updates class/style. |
| Module | Elements > Dynamic Properties |
| Priority | Medium |
| Severity | Major |
| Preconditions | DemoQA is accessible. |
| Test Data | N/A |
| Environment | Web, Chrome latest, macOS; URL: https://demoqa.com/ |
| Postconditions | N/A |

## Test Steps

| Step | Action | Expected Result |
| --- | --- | --- |
| 1 | Open https://demoqa.com/. | Home page loads. |
| 2 | Open the **Elements** section. | Elements page opens. |
| 3 | Click **Dynamic Properties** from the left menu. | Dynamic Properties page opens. |
| 4 | Check the **Will enable 5 seconds** button immediately. | Button is disabled initially. |
| 5 | Check **Visible After 5 Seconds** button immediately. | Button is not visible initially. |
| 6 | Wait for dynamic update. | **Will enable 5 seconds** becomes enabled. |
| 7 | Wait for delayed visibility. | **Visible After 5 Seconds** button becomes visible. |
| 8 | Observe **Color Change** button class/state transition. | Dynamic class/style changes after delay. |
