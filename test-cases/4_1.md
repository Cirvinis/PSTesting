# Test Case: Data-Driven Search and Cart Flow

## Test Case Attributes

| Attribute | Value |
| --- | --- |
| Test Case ID | TC-4-1 |
| Title | Data abstraction for login and search workflow |
| Objective | Validate login/register, search, and add-to-cart behavior using external input data. |
| Requirement | Input data must come from external file and test must execute preconditions and postconditions. |
| Module | Authentication, Search, Cart |
| Priority | High |
| Severity | Major |
| Preconditions | User is registered and logged in; cart is empty. |
| Test Data | `test-data/task4_1-data.json` |
| Environment | Web, Chrome latest, macOS; URL: https://demowebshop.tricentis.com/ |
| Postconditions | User logs out; cart is cleaned up. |

## Test Steps

| Step | Action | Expected Result |
| --- | --- | --- |
| 1 | Load test data from external JSON file. | Data is parsed and available to the test. |
| 2 | Execute precondition: register/login user. | User is authenticated and `Log out` is visible. |
| 3 | Execute precondition: open cart and remove existing items. | Cart contains 0 items. |
| 4 | Search using first term from external data. | Search results page shows at least one result. |
| 5 | Add first result to cart. | Success notification confirms item added. |
| 6 | Search using second term from external data. | Search results page shows at least one result. |
| 7 | Add first result to cart. | Success notification confirms item added. |
| 8 | Open shopping cart. | Cart contains at least 2 line items. |
| 9 | Execute postcondition: remove all cart items. | Cart is empty. |
| 10 | Execute postcondition: log out. | `Log in` link is visible. |
