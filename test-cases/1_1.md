# Test Case: E-Commerce Workflow (Happy Path)

## Test Case Attributes

| Attribute | Value |
| --- | --- |
| Test Case ID | TC-ECOM-001 |
| Title | End-to-end purchase flow with price-filtered items |
| Objective | Validate a complete shopping workflow with navigation, CRUD on cart, and checkout for items priced above $900. |
| Requirement | User can browse, filter by price, add multiple items to cart, update cart, remove item, and complete checkout. |
| Module | Catalog, Cart, Checkout, Account |
| Priority | High |
| Severity | Critical |
| Preconditions | User has a valid account; site is available; test user has valid billing and shipping data. |
| Test Data | User: demo_user@example.com / P@ssw0rd123; Price filter: > 900; Payment: Check/Money Order. |
| Environment | Web, Chrome latest, macOS; URL: https://demowebshop.tricentis.com/ |
| Dependencies | Email inbox not required; tax/shipping rules default. |
| Postconditions | Order confirmation displayed; cart is empty. |
| Notes | Use products with list price > $900 (example: expensive computer, premium laptop). |

## Test Steps

| Step | Action | Expected Result |
| --- | --- | --- |
| 1 | Open the browser. | Browser launches successfully. |
| 2 | Navigate to https://demowebshop.tricentis.com/. | Home page loads. |
| 3 | Click "Log in" in the header. | Login page opens. |
| 4 | Enter a valid email and password. | Credentials are accepted. |
| 5 | Click "Log in". | User is logged in; "Log out" is visible. |
| 6 | Click "Computers" in the top menu. | Computers category page opens. |
| 7 | Click "Desktops". | Desktops subcategory page opens. |
| 8 | Open price filter and set minimum price to 900. | Product list is filtered to items priced above $900. |
| 9 | Open the first product with price > $900. | Product detail page opens. |
| 10 | (CRUD-Create) Add the product to the cart. | Success message appears; cart count increases to 1. |
| 11 | Navigate back to Desktops category. | Desktops list is shown. |
| 12 | Open a second product with price > $900. | Product detail page opens. |
| 13 | Add the second product to the cart. | Success message appears; cart count increases to 2. |
| 14 | Click the shopping cart icon. | Shopping cart page opens. |
| 15 | Verify both items are listed. | Cart shows two distinct items. |
| 16 | (CRUD-Update) Change quantity of the first item to 2. | Cart updates; subtotal recalculates. |
| 17 | Click "Update shopping cart". | Updated quantities persist. |
| 18 | (CRUD-Delete) Remove the second item. | Item disappears; cart count decreases. |
| 19 | Accept terms of service. | Terms checkbox is selected. |
| 20 | Click "Checkout". | Checkout flow starts. |
| 21 | Confirm billing address details. | Billing step completes. |
| 22 | Select shipping address and continue. | Shipping step completes. |
| 23 | Choose shipping method and continue. | Shipping method is accepted. |
| 24 | Choose payment method "Check / Money Order". | Payment method is accepted. |
| 25 | Confirm payment information and continue. | Payment step completes. |
| 26 | Review order summary. | Correct item, quantity, and total are displayed. |
| 27 | Place the order. | Order confirmation page appears with an order number. |
| 28 | Go to "Shopping cart". | Cart is empty. |
