# Test Case: Progress Bar Synchronization

## Test Case Attributes

| Attribute | Value |
| --- | --- |
| Test Case ID | TC-DEMOQA-3-1 |
| Title | Verify Progress Bar synchronization and reset behavior |
| Objective | Validate progress bar state transitions (start, stop, complete, reset) using synchronized checks. |
| Requirement | Progress bar updates correctly, can be stopped/resumed, reaches 100%, and resets to 0%. |
| Module | Widgets > Progress Bar |
| Priority | Medium |
| Severity | Major |
| Preconditions | DemoQA is accessible. |
| Test Data | N/A |
| Environment | Web, Chrome latest, macOS; URL: https://demoqa.com/ |
| Postconditions | Progress bar can be reset to initial state. |

## Test Steps

| Step | Action | Expected Result |
| --- | --- | --- |
| 1 | Open https://demoqa.com/. | Home page loads. |
| 2 | Open the **Widgets** section. | Widgets page opens. |
| 3 | Click **Progress Bar** from the left menu. | Progress Bar page opens. |
| 4 | Verify initial progress value. | Progress value is `0%` (or `aria-valuenow=0`). |
| 5 | Click **Start**. | Progress starts increasing above `0%`. |
| 6 | Click **Stop** while progress is running. | Progress value stops changing. |
| 7 | Click **Start** again and wait for completion. | Progress reaches `100%` and **Reset** appears. |
| 8 | Click **Reset**. | Progress returns to `0%`. |
